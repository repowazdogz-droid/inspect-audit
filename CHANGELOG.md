# Changelog

All notable changes to this project are documented here. The format is loosely
based on [Keep a Changelog](https://keepachangelog.com/).

## [0.1.0] — 2026-07-13

First release.

### Added
- Static, read-only auditor for Inspect `.eval` logs with 24 checks across five
  families: denominator integrity, scorer integrity, judge integrity, run
  integrity, and provider/regime comparability (including cross-run checks).
- `inspect-audit` CLI with `PASS`/`WARN`/`FAIL` verdicts, `--json`, `--verbose`,
  `--header-only`, and `--exit-zero`; verdict shown at the top and bottom.
- Public Python API: `audit_paths`, `AuditReport`, `Status`, `to_dict`,
  `to_json`, and the check `CATALOG`.
- A first-class check catalog as the single source of truth for check metadata,
  with tests enforcing that documented and implemented checks stay identical.
- Committed example logs (`examples/clean.eval`, `examples/broken.eval`) and a
  fixture corpus generated by Inspect's own writer.
- Mutation test per check, plus read-only, determinism, false-positive, error,
  registry, and public-API tests.
- Assurance-boundary and technical write-up docs.

### Guarantees
- Read-only: source logs are never modified (test-enforced).
- Deterministic output: the same log yields byte-identical reports
  (test-enforced).
